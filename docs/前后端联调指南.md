# PHM Platform 前后端联调指南

## 📋 联调前准备

### 1. 环境要求
- **Java**: 17 或更高版本
- **Maven**: 3.8+ 
- **Node.js**: 18+ 和 npm
- **Docker**: 用于启动数据库（可选，也可使用本地数据库）

### 2. 端口占用检查
确保以下端口未被占用：
| 服务 | 端口 | 说明 |
|------|------|------|
| Gateway | 8080 | API网关 |
| Frontend | 5173 | 前端开发服务器 |
| Collection | 8101 | 采集层服务 |
| Computation | 8102 | 计算层服务 |
| Storage | 8103 | 存储层服务 |
| Service | 8104 | 服务层服务 |
| PostgreSQL | 5432 | 时序数据库 |
| Neo4j | 7474/7687 | 图数据库 |
| MySQL | 3306 | 关系数据库 |

## 🚀 启动服务

### 方式一：使用启动脚本（推荐）

```bash
# 在项目根目录执行
start-services.bat
```

脚本会自动按顺序启动：
1. Docker 数据库服务
2. Maven 编译项目
3. Gateway 服务 (8080)
4. 四个数据服务 (8101-8104)
5. 前端开发服务器 (5173)

### 方式二：手动启动

#### 1. 启动数据库（使用Docker）
```bash
docker-compose up -d
```

#### 2. 编译项目
```bash
mvn clean install -DskipTests
```

#### 3. 启动后端服务（每个服务一个终端）
```bash
# 终端1: Gateway
cd phm-gateway
mvn spring-boot:run

# 终端2: 采集层
cd phm-data-service/data-collection
mvn spring-boot:run

# 终端3: 计算层
cd phm-data-service/data-computation
mvn spring-boot:run

# 终端4: 存储层
cd phm-data-service/data-storage
mvn spring-boot:run

# 终端5: 服务层
cd phm-data-service/data-service
mvn spring-boot:run
```

#### 4. 启动前端
```bash
cd phm-frontend
npm install  # 首次运行
npm run dev
```

## 🔌 前后端通信原理

### 架构流程图

```
┌─────────────┐      ┌─────────────┐      ┌─────────────────────────────────────┐
│   前端      │ ──▶  │  Vite代理   │ ──▶  │           Gateway (8080)            │
│  (5173)     │      │  (/api)     │      │  ┌─────────┬─────────┬─────────┐   │
└─────────────┘      └─────────────┘      │  │/collection│/computation│/service│   │
                                          │  │  (8101)  │  (8102)   │ (8104) │   │
                                          │  └─────────┴─────────┴─────────┘   │
                                          └─────────────────────────────────────┘
```

### 关键配置说明

#### 1. 前端 Vite 代理配置 (`vite.config.js`)
```javascript
server: {
  port: 5173,
  proxy: {
    '/api': {
      target: 'http://localhost:8080',  // 所有 /api 请求转发到 Gateway
      changeOrigin: true
    }
  }
}
```

#### 2. Gateway 路由配置 (`application.yml`)
```yaml
spring:
  cloud:
    gateway:
      routes:
        # 采集层路由
        - id: collection-route
          uri: http://localhost:8101
          predicates:
            - Path=/api/collection/**
          filters:
            - StripPrefix=2  # /api/collection/sensor → /sensor
        
        # 计算层路由
        - id: computation-route
          uri: http://localhost:8102
          predicates:
            - Path=/api/computation/**
          filters:
            - StripPrefix=1  # /api/computation/sync → /computation/sync
```

#### 3. 前端 API 封装 (`request.js`)
```javascript
const request = axios.create({
  baseURL: '/api',  // 所有请求以 /api 开头
  timeout: 30000
})

// 采集层 API
export const collectionApi = {
  sendSensorData: (data) => request.post('/collection/sensor', data),
  // 实际请求: POST http://localhost:5173/api/collection/sensor
  // Vite代理转发到: http://localhost:8080/api/collection/sensor
  // Gateway路由到: http://localhost:8101/sensor
}
```

## 🔧 常见问题排查

### 问题1: 前端页面无法访问
```
症状: 浏览器显示无法连接 localhost:5173
排查:
1. 检查前端服务是否启动: cd phm-frontend && npm run dev
2. 检查是否有报错信息
3. 检查端口5173是否被占用: netstat -ano | findstr 5173
```

### 问题2: API请求404错误
```
症状: 浏览器控制台显示 404 Not Found
排查:
1. 检查 Gateway 是否启动: http://localhost:8080/actuator/health
2. 检查后端服务是否启动: http://localhost:8101/health
3. 检查请求路径是否正确（查看 request.js 中的定义）
4. 检查 Gateway 路由配置是否正确
```

### 问题3: 跨域错误 (CORS)
```
症状: 浏览器控制台显示 CORS policy 错误
原因: 前端直接请求后端，没有经过 Vite 代理
解决: 确保前端使用相对路径 /api/xxx，不是完整URL
```

### 问题4: 数据库连接失败
```
症状: 存储层服务启动报错，无法连接数据库
排查:
1. 检查 Docker 是否运行: docker ps
2. 检查数据库端口是否被占用
3. 检查 application.yml 中的数据库配置
4. 手动测试连接: docker exec -it phm-postgres psql -U phm -d phm_db
```

## 📝 调试技巧

### 1. 浏览器开发者工具
- **Network面板**: 查看所有API请求和响应
- **Console面板**: 查看日志和错误信息
- **Vue Devtools**: 查看组件状态和Props

### 2. 后端日志
每个服务窗口都会输出日志，关注：
- `[Request]` 开头的请求日志
- `[Response]` 开头的响应日志
- `ERROR` 级别的错误日志

### 3. 使用curl测试API
```bash
# 测试采集层健康检查
curl http://localhost:8080/api/collection/health

# 测试发送传感器数据
curl -X POST http://localhost:8080/api/collection/sensor \
  -H "Content-Type: application/json" \
  -d '{"deviceId":"EQ-001","sensorType":"temperature","value":25.5}'

# 测试时序数据查询
curl "http://localhost:8080/api/storage/timeseries/query?deviceId=EQ-001&start=2024-01-01T00:00:00Z&end=2024-12-31T23:59:59Z"
```
