# PHM 数据管理平台 - 数据库安装与使用指南

## 一、快速启动（推荐）

### 前提条件
- 已安装 [Docker Desktop](https://www.docker.com/products/docker-desktop)

### 启动数据库服务

```bash
# 1. 进入项目目录
cd D:\1\PHM-Platform

# 2. 启动所有数据库服务（PostgreSQL + Neo4j）
docker-compose up -d

# 3. 查看服务状态
docker-compose ps

# 4. 查看日志
docker-compose logs -f
```

### 访问数据库

| 服务 | 访问地址 | 账号 | 密码 |
|------|---------|------|------|
| **Neo4j 浏览器** | http://localhost:7474 | neo4j | phm123456 |
| **PostgreSQL** | localhost:5432 | phm | phm123 |

### 停止服务

```bash
# 停止所有服务（保留数据）
docker-compose down

# 停止并删除所有数据（谨慎使用！）
docker-compose down -v
```

## 二、手动安装（不使用 Docker）

### 1. PostgreSQL + TimescaleDB 安装

#### Windows 安装步骤

1. **下载 PostgreSQL**
   - 访问：https://www.postgresql.org/download/windows/
   - 下载 PostgreSQL 15.x 安装包

2. **安装 PostgreSQL**
   - 运行安装程序
   - 记住设置的密码（建议设为 `phm123`）
   - 端口保持默认 `5432`
   - 安装 pgAdmin（图形化管理工具，推荐勾选）

3. **安装 TimescaleDB 扩展**
   
   打开 `SQL Shell (psql)` 或 pgAdmin，执行：
   
   ```sql
   -- 创建数据库
   CREATE DATABASE phm_db;
   
   -- 连接数据库
   \c phm_db
   
   -- 安装 TimescaleDB 扩展
   CREATE EXTENSION timescaledb;
   
   -- 验证安装
   SELECT timescaledb_pre_restore();
   ```

4. **执行初始化脚本**
   
   在 pgAdmin 中打开 `init-scripts/01-init-timescaledb.sql`，执行所有 SQL。

#### 修改项目配置

编辑 `phm-data-service/data-storage/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/phm_db
    username: postgres    # 或你设置的用户名
    password: 你的密码
    driver-class-name: org.postgresql.Driver
```

### 2. Neo4j 安装

#### 方法一：Neo4j Desktop（推荐，有图形界面）

1. 访问：https://neo4j.com/download/
2. 下载 Neo4j Desktop
3. 安装后创建新项目
4. 点击 "Add" → "Local DBMS"
5. 设置密码为 `phm123456`（与项目配置一致）
6. 点击 "Start" 启动，然后 "Open" 打开浏览器界面

#### 方法二：Neo4j Community Server

1. 访问：https://neo4j.com/download-center/#community
2. 下载 Neo4j Community Edition
3. 解压到任意目录（如 `D:\neo4j-community-5.x.x`）
4. 配置环境变量或直接运行：

```bash
# 进入 bin 目录
cd D:\neo4j-community-5.x.x\bin

# 安装为 Windows 服务
neo4j.bat install-service

# 启动服务
neo4j.bat start

# 停止服务
neo4j.bat stop
```

5. 浏览器访问：http://localhost:7474
6. 首次登录账号：neo4j，需要修改密码为 `phm123456`

## 三、数据库表结构说明

### PostgreSQL (TimescaleDB) 表

| 表名 | 用途 | 说明 |
|------|------|------|
| `sensor_time_series` | 传感器时序数据 | TimescaleDB 超表，自动按时间分区 |
| `documents` | 文档存储 | 维修记录、FMECA 等文档 |
| `failure_modes` | 失效模式 | FMECA 分析数据 |
| `devices` | 设备信息 | 设备基本信息表 |
| `sensor_hourly_stats` | 小时统计视图 | 自动预计算的统计指标 |

### Neo4j 图结构

| 节点类型 | 属性 | 用途 |
|---------|------|------|
| `Equipment` | equipmentId, name, type, location | 设备节点 |
| `Component` | componentId, name, type | 组件节点 |

| 关系类型 | 含义 |
|---------|------|
| `HAS_COMPONENT` | 设备包含组件 |
| `RELATED_TO` | 组件间关联 |

## 四、常见问题

### Q1: Docker 启动失败，端口被占用

```bash
# 查看端口占用
netstat -ano | findstr :5432
netstat -ano | findstr :7474
netstat -ano | findstr :7687

# 修改 docker-compose.yml 中的端口映射
# 例如将 "5432:5432" 改为 "5433:5432"
```

### Q2: 项目连接数据库失败

1. 检查数据库服务是否启动：
   ```bash
   docker-compose ps
   ```

2. 检查配置文件中的密码是否正确

3. 检查防火墙是否允许端口访问

### Q3: 如何查看/修改数据库数据

**PostgreSQL:**
- 使用 pgAdmin（图形化）
- 使用 DBeaver（通用数据库工具）
- 使用 IDEA 的 Database 工具窗口

**Neo4j:**
- 浏览器访问 http://localhost:7474
- 使用 Neo4j Browser 执行 Cypher 查询

### Q4: 数据存储在哪里

**Docker 方式：**
- 数据存储在 Docker Volume 中，即使删除容器数据也不会丢失
- 彻底删除数据：`docker-compose down -v`

**手动安装方式：**
- PostgreSQL：安装目录的 `data` 文件夹
- Neo4j：安装目录的 `data` 文件夹

## 五、常用操作

### 备份数据

```bash
# 备份 PostgreSQL
docker exec phm-postgres pg_dump -U phm phm_db > backup.sql

# 备份 Neo4j
docker exec phm-neo4j neo4j-admin database dump neo4j --to-path=/backups
```

### 重置数据库（清空所有数据）

```bash
# 停止服务并删除数据卷
docker-compose down -v

# 重新启动（会自动重新初始化）
docker-compose up -d
```

## 六、技术选型说明

### 为什么选择 PostgreSQL + TimescaleDB？

1. **时序数据优化**：TimescaleDB 是 PostgreSQL 的扩展，专门优化时序数据存储和查询
2. **自动分区**：数据自动按时间分区，查询效率高
3. **连续聚合**：自动预计算统计指标，加速常用查询
4. **数据保留策略**：自动清理过期数据，节省存储空间
5. **与项目兼容**：你的项目 pom.xml 已配置 PostgreSQL 驱动

### 为什么选择 Neo4j？

1. **图数据库**：适合存储设备-组件关系和故障传播路径
2. **路径查询**：支持最短路径、关系遍历等图算法
3. **可视化**：自带浏览器界面，方便查看知识图谱
